Aside from the UI, there is one critical difference: S4S edits packages, and VS Code edits individual text files. S4S treats packages as your source code (i.e. files you edit to make your mod), while VS Code with S4TK treats packages as build artifacts (i.e. files that are generated by an automated build process).

Both of these approaches have their pros and cons, and for many mod creators, editing packages directly may still be necessary. Sims 4 Studio is an indespensible modding tool, especially for CC creation; this article is in no way trying to convince you to use one over the other, as they each have their place and can be used together.

Advantages of VS Code with S4TK

The primary draw of Sims 4 Toolkit is that it lets you make mods from loose XML files rather than packages, but why does this matter? Most software development is done with individual text files rather than binary ones for many reasons:

  • You have full control over your project's file structure
  • You're not restricted to programs that can read packages
  • You won't break your entire project if one file is corrupt
  • You can edit one file without reading/writing the entire package
  • You can use Git for source control and back your code up on GitHub

Additionally, the S4TK extension handles resource keys for you. You do not have to include them in file names or set them in an index; S4TK will infer the correct type, group, and instance for tuning / SimData from their contents.

If types, groups, and/or instances cannot be inferred, you can override them with an XML comment.

The best part is, you don't have to write a single line of code to reap these benefits, since the S4TK VS Code extension handles the build script for you. All you have to do is edit a JSON file (the S4TK Config) to tell it which files belong in which packages.

Limitations of VS Code with S4TK

While Sims 4 Toolkit works great for files related to gameplay (tuning, SimData, images, and string tables), it does not support many file types related to CC, such as object catalogs, rigs, and ASMs. For these kinds of files, you must use Sims 4 Studio. However, you can include these packages within your S4TK project's source code to merge them into other built packages.

There are no plans to add support for CC to the S4TK extension, as S4TK itself does not support it either.

Other limitations of the S4TK VS Code extension include its current lack of non-build-related features, such as extracting tuning. For this, you'll either have to use Sims 4 Studio, XML Extractor, TDESC Browser, or @s4tk/extraction.

There are plans to add extraction/file generation abilities to the S4TK extension in the future.
