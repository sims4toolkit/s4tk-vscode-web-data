S4TK has always been intended for making gameplay mods, so it only supports tuning, SimData, and string tables as first-class citizens. However, all other Sims 4 resources can be built as well via packages or TGI file names.

"TGI file names" refers to file names that contain the type, group, and instance, such as 12345678_00000000_0123456789ABCDEF.binary

Tuning & SimData

All loose tuning files must have an .xml extension, and their SimDatas must be in the same folder and have the same name, but with a .SimData.xml extension. If using these extensions, they must be plain-text (i.e., no binary SimDatas).

You are allowed to use TGI file names for tuning and SimData, but it is highly advised against as it will cause issues with intellisense.

String Tables

String tables can be binary or JSON, but JSON is highly recommended. JSON string tables must use the .stbl.json extension, and binary string tables must have TGI file names.

Deleted Records

Deleted records are resources with a flag that tells the game to treat them like they don't exist. You can use deleted records to delete vanilla files, or to deactivate files in your own mod via an add-on.

Deleted records must use a TGI file name with the extension .deleted. For instance, if you want to delete a file with the key 12345678-00000000-0123456789ABCDEF, you can use a file named 12345678_00000000_0123456789ABCDEF.deleted.

Warning: It is generally not a good idea to use deleted records. S4TK only supports them because there is no other tool that exists for them, and I wanted to provide people with a way to do it if they want to.

S4S does not render deleted records, so you can only see them with the package renderer in VS Code or the online S4TK package viewer.

All Other Resources

All other resources, such as DDS images, object definitions, and ASMs must either be kept in packages or written as binary files using TGI file names. Packages are recommended, since you can edit them with S4S as needed.
