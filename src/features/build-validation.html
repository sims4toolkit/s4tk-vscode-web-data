The build script checks for potential warnings and errors before and while the build script is running. If anything is seriously wrong, the build script will terminate with a detailed error message. Whether the build succeeds or not, a file called BuildSummary.json will be output that details how the config was interpreted and which specific files had problems.

Pre-Build Validation

Before the build script runs, it verifies some things:

  • Every package has a non-empty, unique filename
  • The source leads to an existing directory
  • All destinations lead to an existing directory

    • Does not apply if allowFolderCreation is true

  • No two package's glob patterns match the same files

    • Does not apply if allowPackageOverlap is true

  • Every source file is captured by at least one package's glob patterns

    • Does not apply if allowMissingSourceFiles is true

  • Each package's glob patterns match at least one source file

    • Does not apply if allowEmptyPackages is true

If building in release mode, some extra checks are done:

  • The ZIP has a non-empty filename
  • If the ZIP's include paths is non-empty, at least one file is matched

Runtime Validation

While the build script is running, it may terminate with an error if:

  • Any resources are corrupt or could otherwise not be built into their TS4 binary formats (string tables, SimData, etc.)
  • Any tuning/SimData file's resource key could not be determined by TGI file names, S4TK comments, or inference

Build Summary

The BuildSummary.json file is generated at your project's root and contains information that helps you debug your project and/or config.
